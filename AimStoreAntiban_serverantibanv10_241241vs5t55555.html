PRIVACY

Zero-Trace Policy • SessionStorage (RAM-only) Operations

Zero-Trace Principle AimStore-Nexus is designed to minimize persistence and maximize anonymity. The system runs without a backend and does not write permanent application data to your hard drive. Storage Model (SessionStorage / RAM-only) - This application uses sessionStorage to maintain runtime state during the active browser tab session. - Data persists through refresh (F5) and normal navigation while the tab remains open. - When the browser tab is closed, the session is destroyed and all sessionStorage data is wiped by the browser. What is stored (while the tab is open) - Login state (ID + admin flag) - Credits balance (CR) - Inventory items acquired during the session - SOCIAL_NET posts created during the session What is NOT stored - No localStorage is used. - No server-side storage exists. - No long-term identifiers or tracking mechanisms are written by this application. Tab Close Wipe (Explicit) Closing the browser tab will wipe session data including Inventory, Credits, and Posts. This is intentional and is the core of the Zero-Trace policy.
